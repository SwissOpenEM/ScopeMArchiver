    Deployment

    Service Deployment Configuration

    Production

    ### Global
HOST=scopem-openem.ethz.ch  # Host where this instance is hosted

### Backend
OPENEM_BACKEND_IMAGE_NAME=ghcr.io/swissopenem/scopemarchiver-openem-backend # Image used for backend service
OPENEM_IMAGE_TAG=main
API_ROOT_PATH=/api/v1 # Backend server api root path

#### Minio
MINIO_REGION="eu-west-1"
MINIO_RETRIEVAL_BUCKET="retrieval"
MINIO_LANDINGZONE_BUCKET="landingzone"
MINIO_STAGING_BUCKET="staging"

#### PREFECT
PREFECT_VERSION=3.0.4-python3.11 # Prefect version used in all images
PREFECT_LOGGING_LEVEL=DEBUG # Logging level
PREFECT_RUNTIME_IMAGE_NAME=ghcr.io/swissopenem/scopemarchiver-openem-runtime # Image name for containers used to execute flows
PREFECT_CONFIG_IMAGE_NAME=ghcr.io/swissopenem/scopemarchiver-openem-config # Image name for configuration container
PREFECT_WORKER_LTS_VOLUME_NAME=nfs-lts-share  # Name of the Docker volume where the LTS NFS share is mounted
PREFECT_JOB_TEMPLATE=prefect-jobtemplate-prod.json # Production Prefect job template
PREFECT_ARCHIVAL_WORKPOOL_NAME=archival-docker-workpool # Workpool name for archiver jobs
PREFECT_RETRIEVAL_WORKPOOL_NAME=retrieval-docker-workpool # Workpool name for retrieval jobs
LTS_ROOT_FOLDER = /tmp/LTS # Root folder where Docker volume of LTS share is mounted, i.e. root folder of the LTS share

### Scicat Mock
SCICAT_MOCK_IMAGE_NAME=ghcr.io/swissopenem/scopemarchiver-openem-scicat-mock
SCICAT_ENDPOINT=http://${HOST}:89

    Development

    For development, it is useful to override some configuration:

    PREFECT_LOGGING_LEVEL=DEBUG
PREFECT_WORKER_LTS_VOLUME_NAME=lts-mock-volume
PREFECT_JOB_TEMPLATE=prefect-jobtemplate-dev.json
LTS_FREE_SPACE_PERCENTAGE=2
HOST=localhost
SCICAT_ENDPOINT=http://scicatmock:8000

    Note: The lts-mock-volume is a local volume here and not the LTS share.

    Prefect Worker

    docker compose --env-file .production.env --env-file .development.env up -d prefect-archival-worker
docker compose --env-file .production.env --env-file .development.env up -d prefect-retrieval-worker

    Flows

    The flows can be deployed using a container:

    docker compose --env-file .production.env --env-file .development.env run --rm prefect-flows-deployment

    This deploys the flows as defined in the prefect.yaml and requires the secrets set up in the previous step.
